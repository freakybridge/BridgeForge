# 调查报告：BOM/no-BOM 统一策略与模板污染修复

日期：2026-07-08

## 结论

BridgeForge 全 repo 文本文件统一使用 UTF-8 without BOM。禁止在模板、入口、脚本、rule、JSON、memory、文档和 git hook 文本文件开头写入 `EF BB BF`。

## 影响范围

- `templates/codex/memory/MEMORY.md` 与 `templates/codex/memory/_stats.json` 带 BOM，`python -m json.tool templates/codex/memory/_stats.json` 会报 `Unexpected UTF-8 BOM`。
- 字节级扫描发现 Codex 模板侧还有 hooks、rules、scripts、doc、CHANGELOG 等文本文件带 BOM；`.codex` dogfood 副本也存在同类污染。
- Claude memory 模板 `templates/claude/memory/MEMORY.md` 与 `_stats.json` 原本无 BOM；本轮补检测，防止后续混入。

## 修复

- 剥离当前发现的 BOM，只删除文件头三字节，不做正文转码。
- 新增 `encoding_check.py` 到 `templates/claude/hooks/`、`templates/codex/hooks/`、`.claude/hooks/`、`.codex/hooks/`。
- 三份 `.githooks/pre-commit` 接入 encoding 硬闸，检测到任一文本文件以 BOM 开头时 exit 2。
- Claude/Codex 模板与自身 dogfood settings 的 `PostToolUse(Edit|Write|MultiEdit)` 接入 encoding 检查，编辑后提前提示 BOM 污染。
- `tests/harness/run_downstream_fixture.py` 新增 `encoding-no-bom` 回归：源树应无 BOM，fixture 人为加 BOM 后必须被拦截。

## 验证项

- 全 repo 文本文件 BOM 扫描应返回 0 个命中。
- `python -m json.tool templates/codex/memory/_stats.json` 应解析成功。
- `python tests/harness/run_downstream_fixture.py --case encoding-no-bom --case precommit-shebang --case root-precommit` 应通过。
