2026-06-25 产品层瘦身第二类 debate 时发现的**既存 bug**（非本次引入）：

## 冲突实体
- `templates/CLAUDE.md §8`：同一 bug 前 **3** 次代码改动失败、第 **4** 次禁止动手（硬停）。
- `templates/rules/debugging.md §6` T1：同一 bug 连续改代码 **≥2** 次仍未解决即升级。

两个文件给鬼打墙红线写了**不同的数（3 vs 2）**，下游 agent 同时加载会精神分裂。违反 [[meta_rule_design]] §4.3「单值红线不能两处各写一个数」。

## 为何暂未修
debate（`docs/debates_2026-06-25_redline-placement.md`）判定：**统一阈值 = 改变 agent 行为**（试几次才停手），属行为变更，**不该混进"只省体积、不改效力"的瘦身**。用户被问"统一成 2 / 3 / 分层(2预警+4硬停)"时选了**不修改**——阈值这种实际红线数值不在本次瘦身动。

## 将来怎么收口
鬼打墙组的**整组去重暂缓**（含 debugging §6 删与 §8 近义的「立即停手」句 + 加 pointer 指回 §8）——因为去重时 pointer 要写"阈值以 §X 为准"，没统一就没法写。所以**阈值统一 + 鬼打墙组去重打包成一次独立决策**，那次先定阈值（审阅者倾向更严的 2，早停比晚停安全），再连带做 §6 结构去重。doc 分层组已在 v0.30.0 单独落地（它不涉及行为变更）。
