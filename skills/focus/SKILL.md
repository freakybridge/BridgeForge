# 任务锚控制

## 定位与边界

操作 session 级短期状态 `.runtime/focus/anchor.json`。`focus_reminder.py` 负责自动捕获和提醒，本 skill 只处理手动查看、纠正、清除与漂移自检。

## 输入

| 输入 | 动作 |
|------|------|
| 无参数 | 查看任务锚并立即自检 |
| `<文本>` | 将任务锚更新为该文本 |
| `clear` | 删除任务锚文件，等待下一条实质消息重新捕获 |

## 核心流程

1. 读取 `.runtime/focus/anchor.json`。文件不存在时，回复“还没有锚，hook 会在下一条实质消息自动捕获”，然后停止。
2. 按输入执行：
   - 无参数：显示 `anchor` 和 `turns`；对照近期工作判断是否仍属原任务。
   - 带文本：只替换 `anchor`，保留 `session` 与 `turns`，用 UTF-8 和 `ensure_ascii=False` 写回。
   - `clear`：删除整个 `anchor.json`，不得只清空字段。
3. 无参数自检时只给一个结论：
   - `✓ 仍在原任务`
   - `⚠ 疑似漂移 → <前置阻塞 / 附加扩张 / 无关支线> → <spinoff / 内联 / todo / 新任务>`

## 输出与验证

- 更新后回复：`✓ 锚已更新为：<文本>`，并重新读取文件确认字段。
- 清除后回复：`✓ 锚已清，下一条实质消息将重新捕获`，并确认文件已不存在。

## 停止条件

- JSON 无法解析或缺少预期字段：停止写入并报告损坏内容的位置。
- 漂移类型无法确定：标记“未验证”，不要强行改锚。

## 禁止事项

- 禁止把任务锚当长期 memory；长期知识使用 `summary`。
- 禁止在 `clear` 时保留含旧 `session` 的文件。
- 禁止把与“当前主任务是什么”无关的内容写入任务锚。
