# escalate — 急停并求外援

## 定位与边界

这是用户兜底的“停止当前尝试 + 独立研判”按钮，不是继续修复的快捷方式。适用于同一问题已连续尝试至少 3 种方案、长时间无进展或用户反复指出方向不对。

## 输入

- 原始任务和当前期望结果。
- 已尝试方案、每次失败证据、当前假设及其证据强弱。
- `$ARGUMENTS`：用户补充的卡点或待挑战假设。

## 核心流程

### 1. 立即急停并自查

停止当前编码和命令。按以下结构诚实总结：

- 原任务。
- 已试方案及每项失败原因。
- 当前假设与对应实证 / 缺证据。
- 自己判断的卡点。

已试方案少于 3 个时，告诉用户尚未达到鬼打墙阈值，并用单题选择确认是再试一轮还是仍然 escalate；当前回合结束。

### 2. 独立研判

把 external blind-spot review 显式分派给 `review-auditor`。要求它：

1. 挑战至少一个核心假设，并指出反例或证据缺口。
2. 判断既有方案是否共享同一错误前提。
3. 提出 3 个新根因假设并按可能性排序。
4. 给出 1-2 条可执行路径，例如验证新假设、重新读规范、换思路、重做需求确认、转 `debate` 或 `collab`。

独立 agent 不得延续当前实现思路、附和主 agent，也不得直接修改实现代码。

### 3. 呈现报告

整合原任务、失败方案、被挑战假设、新根因排序、证据和推荐路径。明确一条推荐与理由，并列可执行备选。

### 4. 用户拍板硬闸

把推荐、备选和放弃作为一个选择题，当前回合在该题结束。用户未选择前禁止沿任何路径动手。用户说“你决定”时选证据最强的一条，说明理由；该路径再次失败时重新 escalate。

## 输出与收据

- 完整自查和失败证据。
- 独立 agent 标识、被挑战假设及 3 个新假设排序。
- 推荐路径、备选路径和用户选择。
- 若继续，明确下一步唯一验证动作。

## 停止条件

- 急停后禁止继续原方案；必须先完成独立研判和用户选择。
- 独立 agent 不可用时，明确报告缺少外部视角，不伪称已研判。
- 用户选择放弃时终止该问题。

## 禁止事项

- 禁止美化、遗漏或模糊失败记录。
- 禁止跳过自查直接派 agent。
- 禁止让外援直接看当前实现细节并沿原惯性修补。
- 禁止给出“再想想”之类不可执行路径。
- 禁止用户拍板前继续编码或运行修复命令。
