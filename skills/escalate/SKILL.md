# /escalate / $escalate — 急停 + 外部视角研判

**定位**：用户主动按下的"急停 + 求外援"按钮。当 CLAUDE.md / AGENTS.md 鬼打墙觉察规则失效（我没自觉），用户感觉我卡住时强制触发。

## 用法

```
Claude: /escalate
Codex:  $escalate

/escalate 或 $escalate              # 强制研判
/escalate 或 $escalate <补充说明>   # 带额外上下文（如 "我觉得你假设错了"）
```

## 执行步骤

### Step 1：强制总结当前状态

我必须**停下手头活**，不能继续修代码 / 跑命令。诚实输出：

```
## escalate 自查

### 原任务
<用户最初问什么 / 我们要解什么 bug>

### 已试 N 种方案
1. <方案 1> → <失败原因>
2. <方案 2> → <失败原因>
...

### 当前假设 vs 证据
| 假设 | 是否有实证 |
|------|-----------|
| <假设 1> | ✓/✗（凭什么）|

### 我感觉哪里卡住
<诚实说，不要美化>
```

不能跳过，不能模糊。如果"已试方案"少于 3 个，提示用户："还没到鬼打墙阶段，要不要再试一轮再 escalate？"

### Step 2：派独立 Agent 做外部视角研判

调用 Agent tool（独立 context，不被我的思路惯性污染）：

```python
Agent({
  subagent_type: "Explore",
  description: "外部视角研判鬼打墙",
  prompt: """
    [贴 Step 1 的完整自查]
    
    任务：作为局外人评判这个卡壳：
    
    1. 我们的"假设"是否本身错了？
       （挑战 ≥ 1 个最核心的假设，给出反例或证据缺失）
    2. 已试方案是不是都建立在同一个错误前提上？
    3. 真因可能是什么？提 3 个新假设，按可能性排序。
    4. 推荐路径（任选 1-2 条）：
       - 验证某个新假设
       - 派 verification agent 重读 spec
       - 完全换思路
       - 请用户重新描述需求
       - /debate（Claude）/ $debate（Codex）让两 agent 辩
       - /collab（Claude）/ $collab（Codex）拆多子任务分治重做
    
    严格禁止：
    - 不要附和"对，你试得对"
    - 必须挑战 ≥ 1 个我们的假设
    - 不要直接看 Implementation 的代码（避免被污染）
    - 推荐路径必须可执行（不是"再想想"）
  """
})
```

### Step 3：呈现报告 + 推荐

把外援研判 + 推荐路径整合呈现：

```
## escalate 报告

### 原任务
<task>

### 已试 N 种方案（全失败）
1. <方案> → <原因>
...

### 我的盲点（外援指出）
**被挑战的假设**：<具体哪条假设可能错>
**真因新假设**：
  1. <最可能> — 因为 <理由>
  2. <次可能> — 因为 <理由>
  3. <边缘可能> — 因为 <理由>

### 推荐路径
✓ <最优>：<具体做什么>
○ <备选 1>：<具体做什么>
○ <备选 2>：<具体做什么>

要走哪条？
```

### Step 4：等用户拍板（⛔ 硬闸）

> ⛔ **硬契约**：呈现推荐路径后，必须把「走哪条」作为 **AskUserQuestion**（选项 = ✓ 最优 / 各备选 / 放弃）**结束当前回合**；用户未选前**禁止**沿任何路径动手。escalate 本就是被急停叫停的场景——擅自挑"最优"继续 = 直接掉回刚被叫停的鬼打墙。

如果用户说 "你决定"：
- 默认走 ✓ 最优
- 但要明确告诉用户："我选 X，因为 Y。如果走不通会再 /escalate（Claude）/ $escalate（Codex）一次"

## 何时按

- 我连续试 ≥ 3 轮没解 → 你感觉"在打墙"
- 你看出我假设有问题但我不听劝
- 跑了 30 轮没看到进展
- CLAUDE.md / AGENTS.md 鬼打墙自觉规则没触发（我没自察成功）
- 你说了几次"还不对"我还在原地打转

## 与其他 skill 的关系

| | `/debate` / `$debate` | `/escalate` / `$escalate` | `/plan` / `$plan` |
|---|----------|-------------|---------|
| 触发 | 设计有 2 选项难抉择 | **卡住、打墙** | 任务前先研究 |
| 机制 | 双 agent 多轮对抗 | 单独立 agent 研判 | 只读分析 |
| 输出 | 共识方案 | 诊断 + 推荐路径 | 任务列表 |
| 后续 | 直接落地 | **可能转调** `/debate` / `$debate`、`/collab` / `$collab`、`/plan` / `$plan` | 用户审批后实施 |

`/escalate` / `$escalate` 是**触发器，不是终态** — 它的"推荐路径"可能就是建议你打 `/debate` / `$debate` 或 `/collab` / `$collab`。

## 禁止

- ❌ 美化已试方案让自己显得没那么糟（必须诚实）
- ❌ 为了讨好用户挑选简单的"推荐路径"（要真的有理由）
- ❌ 不诚实承认"哪里卡"（卡了就说卡了）
- ❌ 跳过 Step 1 直接派 agent（外援需要完整自查作输入）
- ❌ Step 4 后擅自动手（必须等用户拍板）
