# collab — 分治协作

## 定位与边界

只处理能拆成无依赖并行子任务的大型改动。需求范围由 `confirm` 确认；本 skill 负责研读、拆分、最终确认、并行执行和独立验证。

## 输入

- 与当前任务目标、范围、约束和验收匹配的确认卡。
- `$ARGUMENTS`：大型改动描述。
- 相关代码、文档、测试入口和 agent 并发能力。

## 核心流程

### 1. 确认卡硬闸

检查本轮有效确认卡。没有、不匹配或已过时就调用 `confirm`；返回前禁止研读、创建协作记录或启动 agent。有效卡存在时直接读取，不重复访谈。

### 2. 研读与规划

1. 把 read-only research 显式分派给 `light-explorer`；只在必要事实缺失时索取一项材料。确认卡与实际任务不匹配时返回 `confirm`。
2. 创建 `collabs_YYYY-MM-DD_<topic>.md`：单 feature 放 `doc/1_plan/<feature>/`；跨 feature 放对应主题的 `multi-feature/`；底层重构放 `doc/1_plan/refactor/`。找不到目录时建立清晰主题目录并说明原因。
3. 记录确认卡路径、目标、范围、约束、验收、拆分表和接口约定。
4. 若无法拆成文件不重叠、无依赖的并行任务，停止并报告原因，不硬凑并行组。

### 3. 拆分确认硬闸

给出目标、子任务、负责文件、依赖、并行组和接口约定的短大纲，并只问一个选择题让用户确认或调整。当前回合在该题结束；用户未确认前禁止启动执行 agent。

单任务控制在 3-5 个文件；粒度由依赖和接口边界决定，不重新访谈执行细节。

### 4. 并行执行

1. 把 independent implementation 显式分派给 `implementation-worker`；同一并行组同时启动多个实例，前组全部完成后才启动下一组。
2. 每个任务 prompt 必须包含整体目标、具体要求、接口约定和唯一文件边界。
3. 实时更新协作记录。
4. 子任务失败最多重试 1 次；再次失败即停止并报告用户。

### 5. 整合与独立验证

1. 主 agent 处理 import、注册、配置等串联并更新协作记录。
2. 把 independent review 显式分派给 `review-auditor`，提供整体目标、接口和改动清单，要求它独立读取真实改动或 `git diff`，检查遗漏、回归与接口衔接。
3. 运行相关测试，把命令、断言、覆盖路径和 review 结论写入记录。
4. review 打回的问题在整合阶段修复，不计入子任务重试次数。

## 输出与收据

- 协作记录路径、拆分表和接口约定。
- 各并行组状态与实际改动文件。
- 主 agent 串联检查、测试收据及独立 review 结论。
- 未完成、未验证或需要用户处理的事项。

## 停止条件

- 没有有效确认卡、拆分不可并行或用户未确认大纲时停止。
- 子任务连续两次失败时停止，不继续循环或悄悄换实现方向。
- 发现范围、约束或验收需变化时返回 `confirm`。

## 禁止事项

- 禁止同一并行组的 agent 修改同一文件。
- 禁止未定义接口和文件边界就分派任务。
- 禁止启动执行 agent 后才补用户确认。
- 禁止用主 agent 的文字总结代替独立 review 真实改动。
- 禁止省略协作记录或验证收据。
